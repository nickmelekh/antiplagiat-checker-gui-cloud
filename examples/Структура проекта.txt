Структура проекта

startup.py - стартовый экран:
	запуск основного приложения в режиме онлайн - Dropbox - main_app.py
	запуск основного приложения в режиме оффлайн (с локального диска) - Local - main_app_local.py
	account - проверка подключения к дропбоксу по accesstoken'у. Если успешно, то выводится приветствие
	settings - задается accesstoken для подключения к дропбоксу. Сохраняется по нажатию ОК. хранится в папке config
	scripts - открывает для редактирования и добавления методов сравнения оконную форму текстового редактора (text_editor.py) и передает в него ссылку на файл с методами проверки
	help - открывает справочник по методом сравнения и анализа текстов - экранная форма для просмотра html страницы - html_viewer_loc.py. Страница для открытия лежит в папке /wiki/article1/lev.html
	exit - закрывает приложение
main_app.py - экранная форма основного приложения.
	при открытии подключается к дропбокс и сканирует хранилище по определенным правилам:
	папка курса - controlwork - result - в ней лежит что надо проверять. Программа выберет все файлы с расширением .ipynb и создаст для каждой папки рекурсивно для всех файлов чекбокс для возможности выбора. Так как преполагается проверка контрольных работ, то файлы должны быть пронумерованы в соответствии с количеством работ. Пример, если работ 9, то нумерация от 0 до 8.
 	список методов автоматически подтягивается из фала с методами проверки из check_scripts.py

 	анализ
 	загрузка
 	график
 	сохранение в Excel
 	выход

	переиспользует часть логики main_app.py за исключением формирования блока интерфейса с выбором файлов. По умолчанию показывает все файлы, хранящиеся в папке /local.

	показ содержимого
